# GitHub skills.md guidance

Use this file when the user wants a human-facing description for publishing the skill outside the runtime.

## Goal

Write a concise `skills.md` that a GitHub reader can understand quickly.

It should explain:
- what the skill does
- when to use it
- the default workflow
- what outputs it produces
- what kinds of sources it supports

## Recommended structure

1. Title
2. One-paragraph summary
3. Supported source types
4. Workflow
5. Output format
6. Notes / limitations

## Keep it human-facing

Unlike `SKILL.md`, `skills.md` can be a normal repo-facing document.
Do not overload it with runtime-only instructions.
Keep examples concrete and short.
